# Chat 入口与右键 Shell 操作迭代纪要

日期：2026-06-20  
主题：补齐 Agent Chat 可用性、右键入口和系统 Shell 操作

## 本轮完成

- 增强 Agent Runtime 的工具触发：对“最大文件 / 大文件 / 深入分析 / 路径分析 / 名称查找”等常见自然语言请求预置只读工具观察，降低模型不输出 JSON fallback tool call 的概率。
- Chat 增加自然语言清空对话：输入 `clear`、`clear chat`、`清空对话`、`重新开始对话`、`新话题` 等会清空 UI 消息和 `ChatService` 历史，减少跨话题 token 污染。
- Treemap 右键菜单新增 `询问 AI`，会把当前节点设为 linked entry，并预填聊天输入。
- TreeView 右键菜单新增 `询问 AI`，同样接入 linked entry 与聊天输入。
- Treemap / TreeView 右键统一支持：打开、文件选择打开方式、在资源管理器中显示、复制路径、系统属性。
- `属性` 不再使用应用内自绘属性面板，改为调用 Windows Shell 原生属性窗口。

## 当前行为

- 对文件夹执行 `打开`：使用资源管理器打开该目录。
- 对文件执行 `打开`：使用系统默认程序打开。
- 对文件执行 `选择打开方式...`：弹出 Windows 打开方式选择窗口。
- 对文件或文件夹执行 `属性`：弹出系统原生属性窗口。
- `询问 AI` 不会自动发送，而是预填输入框，用户可编辑后发送。

## 限制

- 工具触发仍然是 JSON fallback + 启发式预置组合，不是 provider 原生 tool calling。
- 右键菜单新增文案目前直接写在代码中，后续可再纳入 resx 本地化。
- Shell 打开/属性依赖目标路径仍存在；扫描后文件被移动或删除时会显示错误提示。

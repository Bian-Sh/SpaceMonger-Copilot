# 2026-06-27 Copilot 用户可见意图修正

## 背景

用户反馈 Copilot 在普通用户询问“都有啥功能、怎么用好你”时暴露了内部工具/函数名；扫描请求“扫描 G 盘，说说我买的游戏”没有直接给确认卡片；身份回答偶发把 SpaceMonger 翻译成“太空漫游者”；App 多语言设置下，AI 回复没有稳定遵守设置语言。

## 本次调整

- 将 `AiSkillRouter` 的模块说明识别扩展到“功能/模块/用好你/what can you do”等泛问，返回面向用户的模块清单，而不是内部工具清单。
- 将中文盘符说法（例如“G 盘”）解析为 `G:\`，使“扫描 G 盘”能直接生成 `StartScan` 确认卡片。
- 身份回答明确保持产品名 `SpaceMonger Copilot`，不翻译为中文名称。
- `ChatViewModel` 接入 `IAiSkillRouter`：
  - 说明类/身份类问题优先使用本地稳定答案，不再依赖模型自由生成。
  - 带扫描/分析动作的请求直接显示确认卡片，不再追问“要不要我建议一个卡片”。
  - 其他需要模型的问题继续把已路由 skill 传给 `ChatService`。
- 更新 `skills/app-guide` 与 `skills/disk-management`：强调普通用户答案不得暴露 `find_by_path`、`propose_copilot_action` 等内部实现名，并遵守 App 语言设置与产品名规则。

## 回归覆盖

新增/更新 `AiSkillRouterTests` 覆盖：

- “说说都有啥功能，我该怎么用好你呢”返回用户可见模块，不包含内部工具名。
- “扫描 G 盘，说说我买的游戏”生成 `G:\` 扫描确认动作。
- App 语言设置优先：设置为中文时英文提问返回中文；设置为英文时中文提问返回英文。

## 验证结果

- `dotnet test tests/SpaceMonger.Core.Tests/SpaceMonger.Core.Tests.csproj --filter AiSkillRouterTests --no-restore`：通过，24/24。
- `dotnet build src/SpaceMonger.App/SpaceMonger.App.csproj -c Debug --no-restore`：通过，有既有兼容性/nullable/重复资源警告。
- `dotnet build tests/SpaceMonger.App.Tests/SpaceMonger.App.Tests.csproj -c Debug --no-restore`：返回失败但输出显示 0 errors，仅有 NU1701 警告；需后续单独排查测试项目/SDK 对 warning-only 退出码的处理。

## 注意

本次未修复无关的既有警告：OpenTK/SkiaSharp 兼容性警告、nullable 警告、`NoItemsSelectedForCleanupToolTip` 重复资源名警告。
## 追加修正：确认卡片执行结果语言

用户截图反馈：英文 App 设置下，确认卡片本体已是英文，但点击 `Start Scan` 后，状态区仍出现中文“扫描完成，已刷新 TreeView、Treemap 和聊天上下文。”。

本次继续修复：

- `MainWindow.Copilot` 中 Copilot 动作执行结果 `AiActionResult.Message/Details` 改为根据 `L.CurrentLanguageName` 输出中/英。
- 扫描完成状态英文为：`Scan complete. TreeView, Treemap, and chat context have been refreshed.`
- 扫描进度详情英文保留/转换为：`Calculating sizes... — <files> files, <folders> folders`。
- 新发布包：`outputs/SpaceMonger-win-x64-folder-20260627-010455/SpaceMonger.App.exe`。
- Computer Use 复验通过：英文设置下点击 `Start Scan` 后，卡片状态不再出现中文。

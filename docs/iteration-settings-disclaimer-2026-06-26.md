# 设置页免责声明板块迭代纪要

时间：2026-06-26

## 背景

本次只实现“设置页面新增免责声明板块”，用于在用户使用扫描、AI 分析、AI 对话和清理能力前，明确隐私说明、数据风险、责任划分和建议操作。

## 改动内容

- 在设置页新增“免责声明”导航项，位置放在设置页面底部，跟随现有设置页滚动和高亮逻辑。
- 新增 `DisclaimerSettingsSection`，复用现有设置卡片背景、边框、圆角、字体和文本层级。
- 免责声明为静态展示内容，不绑定、不保存任何设置状态，不影响 API Key、模型、删除模式、主题等现有配置项。
- 更新中英文本地化资源：
  - `Strings.zh-CN.resx`：中文文案更完整，覆盖隐私、风险、责任、建议操作。
  - `Strings.resx`：英文文案保持等价含义。

## 验收关注

- 设置页底部可看到“免责声明”板块。
- 中英文资源文件仍可作为 XML 正常解析。
- 免责声明不参与自动保存逻辑。
- 后续通过 `dotnet test` 与发布产物验证整体构建状态。

## 文案二次调整

根据反馈，移除了“隐私说明：”“数据风险：”“责任划分：”“建议操作：”这类段落前缀，改为更接近常见应用免责声明的自然段落表达。新版文案强调辅助判断属性、用户主动选择路径、启用 AI 时可能发送的必要上下文、第三方模型服务条款约束、误判风险、清理前核对与备份建议，语气保持正式但不过度恐吓。
